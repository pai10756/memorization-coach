# 課文背誦小教室 — 設計方法論

這份文件記錄這套「課文背誦小教室」的**設計原則**：背後的記憶科學、五個互動模式的存在理由、視覺與技術約束。
任何新課文（詩、散文、文言文、英文段落）的 HTML，都應遵循這份方法論。

> 換課文時要填什麼，看 [PROMPT_TEMPLATE.md](PROMPT_TEMPLATE.md)。
> 這份文件回答「**為什麼**這樣設計」，模板回答「**該填什麼**」。

---

## 一、記憶原理（Why — 不可妥協的科學基礎）

每一個互動模式的存在都對應一條原理。如果未來想增刪模式，先檢查它對應哪一條，沒有就不要加。

### 1. 提取練習（Retrieval Practice）
**主張**：不是一直看，而是把書蓋起來自己想。測驗本身不只檢查記憶，更會增強長期記憶。
**來源**：Roediger & Karpicke（2006）— *Test-Enhanced Learning*。
**對應模式**：遮蔽測試、打亂順序挑戰、闖關計時。

### 2. 間隔練習（Spaced Practice）
**主張**：不要一天硬背很久，要分散到多天。分散練習通常比集中練習更有利於長期保留。
**來源**：Cepeda et al.（2006）— 數百個實驗的綜述。
**對應模式**：闖關計時的 15 分鐘段落式設計（鼓勵每天一輪，而非一次背完）。
**設計含意**：localStorage 儲存進度，方便明天接續。

### 3. 雙重編碼（Dual Coding）
**主張**：文字加畫面，比只有文字更容易記。語言系統 + 圖像系統並用，記憶更牢。
**來源**：Paivio 的雙重編碼理論。
**對應模式**：故事模式、逐句／逐段學習（每句／每段都搭一張水彩風圖）。
**設計含意**：圖片不是裝飾，是記憶錨點 — 圖必須與內容**語義對應**，不能只是漂亮的背景。

### 4. 降低認知負荷（Cognitive Load）
**主張**：小五學生工作記憶容量有限，一次塞太多會背不起來；長文要切成小單位。
**來源**：Sweller 的認知負荷理論。
**對應模式**：逐句學習（一次只看一句）、闖關分段（一次背 2 句而非 4 句）。
**設計含意**：不要在同一畫面同時呈現所有資訊；用模式切換、進度條、解鎖機制控制節奏。

### 5. 有效學習技巧排序（What Actually Works）
**主張**：在眾多學習技巧中，「練習測驗」與「分散練習」最具實用價值。
**來源**：Dunlosky et al.（2013）— *Improving Students' Learning With Effective Learning Techniques*。
**設計含意**：把最多畫面、按鈕、回饋給「測驗類」模式（Test、Quiz、Challenge），故事與學習模式只是序章。

## 二、五個核心互動模式（必備）

每篇課文的 HTML 至少要有以下五個模式。它們對應上述五條原理，缺一個就少一條科學依據。

### 1. 故事模式（Story Mode）
- **目的**：建立全文輪廓 + 雙重編碼。
- **內容**：所有句子／段落的圖片橫排（手機直排），每張圖下方標題 + 4 字標籤。
- **互動**：點圖展開該句／段的詳細卡片（關鍵詞、白話、記憶提示）。
- **科學**：雙重編碼。

### 2. 逐句／逐段學習（Learn Mode）
- **目的**：深入學習單一單位，控制資訊量。
- **內容**：一次只顯示一句／一段的完整資訊（大圖 + 關鍵詞 + 白話 + 記憶提示）。
- **互動**：上一個 / 下一個按鈕、進度條。
- **科學**：降低認知負荷。

### 3. 遮蔽測試（Test Mode）
- **目的**：強迫提取，從圖回想文字。
- **內容**：圖片 + 被遮蔽的句子。
- **三段難度**（必備）：
  - Level 1：顯示前段，遮後段（例：「雲鬢花顏___」）
  - Level 2：只顯示首字（例：「雲______」）
  - Level 3：完全遮蔽（_______）
- **互動**：點圖揭曉答案；答對／答錯按鈕讓使用者自評。
- **科學**：提取練習。

### 4. 打亂順序挑戰（Quiz Mode）
- **目的**：避免「靠順序背」，測語意理解。
- **內容**：用語意問題（不照原文順序）問「哪一句／段在寫 ___？」，提示首字。
- **互動**：使用者點選或輸入答案，立即顯示對錯與正解。
- **科學**：提取練習 + 防止「序列依賴記憶」。

### 5. 闖關計時（Challenge Mode）
- **目的**：把分散練習壓進一節課的時間盒。
- **內容**：總長 ~15 分鐘的倒數計時器，分 5 關依序解鎖。
- **典型關卡配置**（可依課文長度調整）：
  - 關卡 1（3 分鐘）：看懂故事，讀 3 次
  - 關卡 2（3 分鐘）：遮住背前半
  - 關卡 3（3 分鐘）：遮住背後半
  - 關卡 4（4 分鐘）：合體背誦
  - 關卡 5（2 分鐘）：看首字提示完整背一次
- **互動**：每關「完成」按鈕；全部完成顯示恭喜畫面。
- **科學**：間隔練習（時間盒）+ 提取練習（每關都背）+ 認知負荷（先背半再合）。

## 三、可選擴充模式（散文／長文用）

詩有節奏與字數規律，5 個模式就夠。**散文／長段落**因為單位長、結構鬆，可加：

### 6. 首頁／導讀（Home）
- 課文長到一個畫面塞不下時，需要一個入口頁說明使用順序、推薦動線。

### 7. 關鍵詞骨架（Skeleton Mode）
- 把每段抽成「關鍵詞列表」，先記骨架再記細節。
- 對應原理：分塊（chunking）+ 降低認知負荷。

### 8. 家長陪背（Parent Mode）
- 列出每段的提問清單、對話範例、判分標準，讓家長／老師可以口頭抽考。
- 對應原理：提取練習（由人施測，比自測更嚴格）。

> 加任何第 6+ 個模式時，必須能說出它對應哪一條科學原理；不能因「看起來酷」就加。

## 四、視覺風格原則

### 4.1 主題色：與課文情境呼應（不固定色票）
不要把所有課文做成同一套顏色。每課應依主題選一組調色：

| 課文性質 | 建議主色調 | 範例 |
|---|---|---|
| 唐詩、古典 | 米白紙色 + 朱紅印章 + 金色 | 《長恨歌》 |
| 自然、海洋、科普 | 米白 + 海藍 + 珊瑚橘 + 沙色 | 《海洋的殺手》 |
| 戰爭、歷史、悲壯 | 灰白 + 墨黑 + 暗紅 | （未來） |
| 田園、寧靜 | 米白 + 抹茶綠 + 木褐 | （未來） |

**不變的部分**（所有課文都要遵守）：
- 主背景偏米白／紙色（#F5EFE0 或近似），降低長時間閱讀的眩光。
- 文字用深褐色系（#3D2817 或近似），不用純黑（純黑在米白背景上對比太硬）。
- 用 1 個強調色（朱紅、海藍、墨黑等）標記「正在啟用 / 重要」。
- 用 1 個輔助色（金色、珊瑚橘、抹茶綠等）標記「次重要 / 點綴」。

### 4.2 字體
- **標題、課文本身**：襯線體 — `Noto Serif TC`、`Source Han Serif TC`、`PingFang TC`、`Microsoft JhengHei`。
- **內文、UI 按鈕**：無襯線 — `Noto Sans TC`、`PingFang TC`、`Microsoft JhengHei`、`system-ui`。
- 不依賴 Google Fonts CDN（要可離線），用使用者系統內建字體即可。

### 4.3 字級下限（重要 — 給小五生用）
- 課文本身（詩句／散文段落）：手機 ≥ 24px，桌面 ≥ 32px。
- 按鈕最小高度 42–44px（適合手指點擊）。
- 內文 ≥ 16px。

### 4.4 過渡與回饋
- 模式切換要有平滑動畫（淡入 + 微上移，0.28–0.32s）。
- 按鈕點擊要有明顯反饋（縮放、位移、底色變化）。
- 答對／答錯有清楚顏色（綠 / 紅）與文字提示。

### 4.5 整體調性
- 適合小五生：不要太花俏、無冗長動畫、無分心元素。
- 但要有**該課文的韻味**（古典、海洋、田園……），不能像通用記帳 App。

## 五、技術要求（不可妥協）

### 5.1 單檔可攜
- **單一 HTML 檔**，所有 CSS、JS、圖片 base64 全部內嵌。
- 傳給家長、學生、老師時，一個檔案丟過去就能用。
- **不依賴外部 CDN**（離線也要能跑）。

### 5.2 框架
- **純 vanilla JS**，不引入 React、Vue、jQuery 等框架。
- 理由：減少體積、減少學習門檻、長期可維護（10 年後還能打開）。

### 5.3 響應式
- **Mobile-first**：手機是主要使用情境（小五生／家長手邊裝置）。
- 桌面版也要好看（≥720px 啟用多欄佈局）。
- 斷點建議：`max-width: 640px` 收成兩欄導覽，`min-width: 720px` 展開為多欄。

### 5.4 無障礙與可用性
- 所有 `<button>` 而非 `<div onclick>`。
- 模式區塊用 `<section aria-labelledby="...">` 標記。
- 文字對比度足夠（米白底 + 深褐字應 ≥ 7:1）。

### 5.5 進度持久化
- 使用 `localStorage` 儲存：
  - 目前停留在哪個模式
  - Learn Mode 看到第幾句
  - Test Mode 自評記錄
  - Challenge Mode 完成關卡
- key 用 `memorization-{{課文識別碼}}-...` 避免不同課文互相覆蓋。

### 5.6 圖片
- 用 AI 生成水彩風或符合課文情境的插圖。
- **每張圖必須語義對應該句／段**，不只是裝飾。
- 風格在同一課內必須統一（同一套調色、同一種畫風）。
- 以 base64 嵌入 HTML（`<img src="data:image/...;base64,...">`）。
- 不寫實照片風。

## 六、檔案命名與發佈

### 6.1 檔名規則
- 主課文（最常用、想當網站首頁的那個）：`index.html`
- 其他課文：`{{課文英文/拼音識別碼}}.html`，例如 `chang-hen-ge-2.html`、`ocean-killer.html`

### 6.2 GitHub Pages
- 根網址（`https://{{user}}.github.io/{{repo}}/`）會載入 `index.html`。
- 其他課文用完整路徑訪問。
- 想做課文選單頁時，把 `index.html` 改成清單頁，連結到各課的 HTML。

## 七、品質檢查清單（每做一課，走一次）

開新課文 HTML 完成後，逐項自我檢查：

- [ ] 五個核心模式齊全（Story / Learn / Test / Quiz / Challenge）
- [ ] Test Mode 有 3 段難度
- [ ] Quiz Mode 的問題不照原文順序
- [ ] Challenge Mode 倒數計時器可用，5 關可解鎖
- [ ] 課文字級：手機 ≥ 24px，桌面 ≥ 32px
- [ ] 按鈕高度 ≥ 42px
- [ ] 主題色與課文情境呼應，不是直接套上一課的色
- [ ] 圖片語義對應，不是裝飾
- [ ] 單檔可開（雙擊 HTML 即可，無需網路）
- [ ] 手機與桌面都測過
- [ ] localStorage 進度可保存與還原
