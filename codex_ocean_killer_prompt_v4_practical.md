# 《海洋的殺手》七段背誦攻略小教室 — Codex Prompt v4 實戰版

> **用途**：給 ChatGPT Codex 從零生成「單檔 HTML 互動式學習工具」  
> **主題**：台灣國小五年級國語課文《海洋的殺手》  
> **核心目標**：家長每天陪孩子用 15–20 分鐘，完成「當天一段」的理解、背誦骨架建立、原文還原與提取練習。  
> **圖片生成**：請直接使用 **GPT Image 2 / gpt-image-2** 生成每一段的「多格漫畫式記憶圖」，再將圖片轉成 base64 內嵌於 HTML。  
> **最重要規則**：七段都必須有「多格漫畫」，不能只用單張概念圖。

---

## 一、請 Codex 先理解：這不是一般閱讀網頁

這是一個給「台灣小五學生 + 家長」放學後使用的背誦工具。

學校的真實要求通常是：

- 每天背一段課文。
- 共七段，分七天背完。
- 老師可能會抽背原文，也可能要求孩子說出大意、關鍵詞或回答課文理解題。

所以這個工具不是要推翻學校要求，而是要用更科學的方法幫孩子達成：

1. **先看懂每段在說什麼。**
2. **再用多格漫畫記住順序。**
3. **再用短骨架與關鍵詞提取。**
4. **最後逐步還原課本原文。**

如果 UI、互動或圖像呈現有模糊地帶，請優先選擇：

> 能讓孩子主動回想、照畫面順序背出課文、家長容易陪背的方案。

不要只追求漂亮，不要做成純展示型網站。

## 二、學習科學設計原理

請在工具設計中落實以下原理：

| 原理 | 實作方式 |
|---|---|
| Dual Coding 雙重編碼 | 每段都有多格漫畫 + 文字骨架，讓孩子同時用畫面與語言記憶 |
| Chunking 組塊化 | 每段壓縮成短標題、記憶提示、短骨架、完整骨架 |
| Retrieval Practice 提取練習 | 用遮蔽、填空、排序、自背、問答取代單純重複閱讀 |
| Spaced Practice 分散練習 | 設計七天進度，每天主背一段，也能複習前面段落 |
| Fading Prompts 漸退提示 | 畫面 → 短骨架 → 關鍵詞 → 句首提示 → 原文 |
| Generative Effect 生成效應 | 鼓勵孩子用自己的話先說，再貼回課文句子 |
| Cognitive Load 控制認知負荷 | 長段落拆成 6 格，短段落也至少拆成 4 格，不要整段一次丟給孩子 |

## 四、全文七段總骨架

請在首頁、故事總覽、段落排序、自己說一遍模式中呈現這條總路線：

美麗小島有垃圾  
→ 黃色小鴨解開謎團  
→ 塑膠微粒進入食物鏈  
→ 塑膠成為海洋殺手  
→ 塑膠大量使用又難分解  
→ 人類必須源頭減塑  
→ 海洋發出警訊

七段短標題固定為：

1. 美麗小島
2. 小鴨洋流
3. 塑膠微粒
4. 海洋殺手
5. 塑膠災難
6. 源頭減塑

議論文結構請標示為：

提出疑問  
→ 案例揭密  
→ 對人與食物鏈的危害  
→ 對海洋生物的危害  
→ 量化問題規模  
→ 提出解決方案  
→ 呼籲每個人行動

## 五、最重要的圖片規則：七段都要多格漫畫

本工具的核心記憶方法是：

> 多格漫畫 + 原文骨架 = 雙重記憶提取

因此：

- 七段都不能只用單張概念圖。
- 每段都必須做成 storyboard comic panel。
- 每一格對應一個背誦節點。
- 孩子看第 1 格、第 2 格、第 3 格，就能照順序想起課文。
- 即使是第七段這種短段落，也至少要 4 格。
- 第二段、第三段、第四段、第五段、第六段至少 6 格。
- 每一段的漫畫都請使用 **GPT Image 2 / gpt-image-2** 生成。
- 每張生成圖請轉成 base64 內嵌 HTML，讓 HTML 可離線開啟。
- 若 Codex 無法直接完成 GPT Image 2 生成與 base64 內嵌，請先建立完整 HTML 架構，並用 `IMAGE_SECTION_1_BASE64_PLACEHOLDER` 到 `IMAGE_SECTION_7_BASE64_PLACEHOLDER` 佔位，不要破壞所有互動功能。

圖片統一風格：

- 水彩風
- 教育漫畫
- 海洋環保主題
- 分格清楚
- 兒童友善
- 不恐怖、不血腥
- 色調：海洋藍、沙灘米、柔和警示紅、珊瑚橘
- 每格可以有極少量中文標題，但不要放太多小字，避免圖片文字錯誤
- 圖片上的文字若無法穩定生成，請把文字放在 HTML 圖片下方的 caption，而不是硬塞進圖片裡

## 六、七段多格漫畫規格

### 第 1 段：5 格漫畫｜美麗小島

1. 南太平洋亨德孫島  
   畫面：藍綠色海水中的小島。  
   背誦提示：南太平洋的亨德孫島。

2. 雪白沙灘與豐富生態  
   畫面：白沙灘、珊瑚、魚群、鳥類。  
   背誦提示：雪白沙灘環繞，擁有豐富美麗的生態。

3. 世外桃源與珊瑚環礁  
   畫面：天堂般的小島、珊瑚環礁。  
   背誦提示：彷彿世外桃源，是少數僅存的珊瑚環礁。

4. 然而：無人小島竟有垃圾  
   畫面：畫面一半美麗，一半出現塑膠垃圾。  
   背誦提示：然而，無人小島竟累積垃圾。

5. 十八公噸與來源之謎  
   畫面：海灘上堆著塑膠瓶、塑膠袋，旁邊有問號。  
   背誦提示：十八公噸垃圾，究竟從哪裡來？

### 第 2 段：6 格漫畫｜小鴨洋流

1. 1992 黃色小鴨事件  
   畫面：黃色小鴨事件卡。  
   背誦提示：西元一九九二年，黃色小鴨事件解答謎團。

2. 貨輪遇風暴  
   畫面：貨輪在北太平洋遇上風暴，小鴨掉海。  
   背誦提示：將近三萬隻黃色塑膠鴨落海。

3. 北上漂流  
   畫面：小鴨沿藍色箭頭往北。  
   背誦提示：北極海、美國東岸、英國。

4. 南下漂流  
   畫面：小鴨沿藍色箭頭往南。  
   背誦提示：夏威夷、印尼、澳洲、南美洲。

5. 洋流高速公路  
   畫面：洋流像高速公路，不斷循環。  
   背誦提示：洋流讓塑膠旅客漂游各地。

6. 海面深海都占領  
   畫面：海面有垃圾，深海也有塑膠。  
   背誦提示：不只是海面，連深海也被塑膠大軍占領。

### 第 3 段：6 格漫畫｜塑膠微粒

1. 日照與風浪拍打塑膠  
   畫面：太陽與海浪讓塑膠瓶碎裂。  
   背誦提示：塑膠受到日照與風浪拍打。

2. 分解成塑膠微粒  
   畫面：放大鏡看見塊狀、細絲、球體狀微粒。  
   背誦提示：分解成極小的塑膠微粒。

3. 微粒更毒  
   畫面：微粒吸附污染物與細菌。  
   背誦提示：釋放有毒物質，吸附污染物與細菌。

4. 浮游生物吃下  
   畫面：浮游生物吞入塑膠微粒。  
   背誦提示：微粒體積小，常被浮游生物食入。

5. 馬里亞納海溝與塑膠鉤蝦  
   畫面：深海剖面、塑膠鉤蝦。  
   背誦提示：最深的馬里亞納海溝也有塑膠微粒。

6. 食物鏈回到人類  
   畫面：蝦 → 小魚 → 大魚 → 海鮮餐桌。  
   背誦提示：大魚吃小魚，人類吃海鮮時到底吃進了什麼？

   背誦提示：吞食塑膠的海鳥。

4. 海龜被魚網困住  
   畫面：海龜被漁網纏住但不血腥。

5. 珊瑚礁與鯨魚受害

6. 800 種生物受威脅

1. 塑膠便宜耐用

6. 垃圾來源

   背誦提示：主要來自河川、海岸、船隻漁具與一次性塑膠製品。

## 八、互動模式設計

請設計成 single-page application，頂部固定導覽列，可切換以下九個模式。

### 1. 首頁 Home

首頁包含：

#### 今日進度看板

- 顯示「今天該背第 X 段」。
- 預設第 1 段。
- 可手動選擇今天要背第幾段。
- 顯示已完成段落 checkmark。
- 有「我要背今天那一段」按鈕，點擊後直接跳到 Restore Mode 並選中今日段落。

#### 今日背誦流程

1. 看今天這段的多格漫畫。
2. 看白話意思。
3. 背短骨架。
4. 看句首提示接原文。
5. 做關鍵詞填空。
6. 完整遮蔽背一次。
7. 家長用問題抽問。

#### 給家長的話

首頁下方加入可展開 / 收合的「給家長的話」。

內容：

這份工具不是要孩子一開始就逐字硬背整篇七段課文。  
我們先讓孩子知道每段在講什麼，再用多格漫畫建立畫面順序，最後逐步貼回課本原文。

建議孩子卡住時，不要直接給答案。  
請照以下順序提示：

畫面 → 短骨架 → 關鍵詞 → 句首字 → 原文

### 2. 七段故事總覽 Story Mode

- 顯示七段流程圖。
- 每段一張卡片：
  - shortTitle
  - memoryHint
  - 多格漫畫縮圖
  - 該段在議論文中的功能
- 七段用箭頭串起來。
- 點擊任一段展開：
  - 原文
  - 白話
  - 關鍵詞
  - 短骨架
  - 完整骨架

### 3. 逐段精讀 Read Mode

- 一次顯示一段。
- 上方顯示第 X / 7 段與進度條。
- 中間顯示該段 GPT Image 2 多格漫畫。
- 下方顯示：
  - 原文
  - 白話摘要
  - 關鍵詞 chips
  - 記憶提示
  - 短骨架
  - 完整骨架
- 重要數字與關鍵詞高亮。
- 有上一段 / 下一段。
- 有「我已讀懂」按鈕，寫入 localStorage。

### 4. 多格漫畫記憶 Comic Mode

這是本工具重點。

- 每段顯示 GPT Image 2 生成的多格漫畫大圖。
- 圖下方用 HTML 顯示每一格對應的：
  - panelTitle
  - memoryText
  - 對應原文片段
- 每個 panel caption 可點擊展開 / 收合。
- 孩子可以只看漫畫與 caption 來回想課文。
- 支援「只顯示圖片」、「顯示提示」、「顯示原文片段」三種狀態。

### 5. 原文還原 Restore Mode

這是最重要的背誦模式。

每段提供四個 Level：

#### Level 1：看短骨架說大意
只顯示 shortSkeleton，讓孩子先用自己的話說。

#### Level 2：句首提示
顯示 sentenceStarters，讓孩子接原文。

#### Level 3：關鍵詞填空
把 originalText 中的重要數字與關鍵詞挖空。

#### Level 4：完整遮蔽
只顯示段落標題 + 多格漫畫，讓孩子完整背一次。

功能：

- 選擇段落。
- 切換 Level。
- 揭曉原文。
- 我會了 / 還要練。
- 自評結果存 localStorage。
- 若孩子按「還要練」，請顯示：
  - 不急，先看漫畫第 1 格開始。
  - 你不是背不起來，只是這一句還沒黏牢。

### 6. 遮蔽測試 Test Mode

請根據段落長度自動調整遮蔽難度：

#### 短段落：少於 100 字
例如第七段。

- Level 1：遮蔽關鍵動詞、數字或核心名詞。
- Level 2：只保留句首 3–5 字。
- Level 3：只顯示短骨架與漫畫。

#### 中段落：100–200 字

- Level 1：遮蔽關鍵詞與重要數字。
- Level 2：每句只保留開頭 4–6 字。
- Level 3：只顯示短版骨架。

#### 長段落：超過 200 字
例如第二段、第三段。

- Level 1：保留主要連接詞與順序，遮蔽關鍵資訊。
- Level 2：只顯示每個意思單位的句首。
- Level 3：只顯示短版骨架與漫畫。

每段要有：

- 揭曉答案
- 答對
- 還要練
- 答對率
- 跳到逐段精讀複習

### 7. 關鍵數據與問答 Quiz Mode

至少 16 題，題型分布：

- 選擇題：至少 5 題
- 填空題：至少 5 題
- 排序題：至少 2 題
- 看提示猜段落：至少 2 題
- 簡答 / 自我複述題：至少 2 題

每題包含：

- question
- type
- options（選擇題）
- answer
- explanation
- relatedSectionId
- reviewButton

答錯時不要只說「答錯了」。

答錯時顯示：

1. 正確答案
2. 來自第幾段
3. 簡短說明
4. 「跳到這段複習」按鈕
5. 溫和鼓勵語

鼓勵語範例：

- 沒關係，這題是在幫你找出還沒黏牢的地方。
- 你已經知道方向了，我們回到那一段再看一次。
- 答錯不是失敗，是提醒你哪個關鍵詞要再練。

建議題庫：

1. [選擇] 亨德孫島位在哪裡？答案：南太平洋
2. [填空] 第一段說亨德孫島彷彿＿＿＿＿。答案：世外桃源
3. [填空] 亨德孫島面積大約只有＿＿平方公里。答案：三十七
4. [填空] 亨德孫島竟累積了＿＿公噸塑膠垃圾。答案：十八
5. [選擇] 黃色小鴨事件發生在哪一年？答案：一九九二年
6. [填空] 將近＿＿隻黃色塑膠鴨在北太平洋落海。答案：三萬
7. [選擇] 課文把洋流比喻成什麼？答案：高速公路
8. [排序] 貨輪風暴 → 小鴨落海 → 隨洋流漂流 → 解答垃圾謎團
9. [填空] 塑膠受到日照與風浪拍打，會分解成＿＿＿＿。答案：塑膠微粒
10. [選擇] 哪個深海地點也發現塑膠微粒？答案：馬里亞納海溝
11. [填空] 科學家命名為「＿＿＿＿」。答案：塑膠鉤蝦
12. [選擇] 第四段說受威脅生物高達多少種？答案：八百種
13. [填空] 每年約有＿＿公噸塑膠垃圾流進海洋。答案：一千一百萬
14. [填空] 到二〇四〇年可能增加到＿＿公噸。答案：兩千九百萬
15. [看提示猜段落] 「源頭減塑、塑戰塑決、淨灘、自備環保用品」是哪一段？答案：第六段
16. [簡答] 為什麼最後說未來可能撿到的不是貝殼，而是寶特瓶瓶蓋？

### 8. 段落排序 Order Mode

- 把七段 shortTitle 打亂。
- 讓孩子用「點選加入順序」方式排回原順序。
- 若你能穩定實作拖拉，也可以用 drag-and-drop，但不要犧牲穩定性。
- 完成後顯示：
  - 是否正確
  - 正確順序
  - 為什麼是這個順序

說明文字請用小五可懂語氣：

先提出問題，再用黃色小鴨解開謎團，接著說塑膠微粒和海洋生物的危害，再用數字說明問題很嚴重，最後提出源頭減塑與行動呼籲。

### 9. 自己說一遍 Retell Mode

- 顯示七段短標題與短骨架。
- 不顯示完整原文。
- 有 3 分鐘計時器。
- 引導孩子看著標籤，用自己的話講一遍整篇。
- 每段有「展開提示」按鈕，可看到 3–5 個關鍵詞。
- 不做嚴格對錯判斷。
- 完成後顯示：
  你已經能用自己的話說出文章骨架，接下來再慢慢貼近課文原文。

### 10. 家長陪背 Parent Mode

每段顯示：

- 段落標題
- 多格漫畫縮圖
- 家長提問
- 孩子卡住時的提示順序
- 句首提示
- 白話轉原文對照 parentToOriginalCheatsheet
- 鼓勵語

提示順序固定：

1. 先給畫面提示。
2. 再給短版骨架。
3. 再給關鍵詞。
4. 再給句首提示。
5. 最後才看原文。

鼓勵語：

- 你已經抓到這段的方向了。
- 先不用一字不差，先把骨架說出來。
- 卡住很正常，我們看關鍵詞再接回去。
- 你不是背不起來，只是這一句還沒黏牢。
- 先背故事線，再補課本原文。
- 這一段你已經比剛剛順很多了。
- 很好，現在我們把白話換回課本句子。
- 不急，先從第一個畫面開始。
- 你已經完成一大半了。
- 再試一次，這次只看提示卡。

### 11. 15 分鐘闖關 Challenge Mode

設計 15 分鐘倒數計時器，分成七關。

- 關卡 1｜2 分鐘：看七段故事總覽。
- 關卡 2｜2 分鐘：背第一段骨架。
- 關卡 3｜3 分鐘：背第二段骨架。
- 關卡 4｜3 分鐘：背第三段骨架。
- 關卡 5｜2 分鐘：背第四、第五段骨架。
- 關卡 6｜2 分鐘：背第六段骨架。
- 關卡 7｜1 分鐘：背第七段結尾警訊。

每關有：

- 關卡說明
- 該關短骨架
- 完成按鈕
- 下一關解鎖

全部完成後顯示：

你已經完成《海洋的殺手》七段背誦攻略！

## 十一、技術要求

- 單一 HTML 檔案。
- CSS 內嵌在 `<style>`。
- JavaScript 內嵌在 `<script>`。
- 不使用 React、Vue、Svelte。
- 使用 Vanilla JavaScript。
- 不依賴外部 CDN、API、圖片、字體或套件。
- 可直接用瀏覽器開啟。
- 支援手機、平板、桌面。
- 使用 localStorage 儲存：
  - 今日段落
  - 已讀懂段落
  - 原文還原自評
  - 測驗答對率
  - 闖關進度
  - 最後使用模式
- 若圖片 base64 太大，可以先使用 placeholder，但 HTML 結構必須完整且互動可用。

## 十二、驗收標準

請確保完成的 HTML 滿足以下檢查清單：

□ 所有七段內容都正確呈現，且 originalText 與本 prompt 完全一致。  
□ 七段 shortTitle 分別是：美麗小島、小鴨洋流、塑膠微粒、海洋殺手、塑膠災難、源頭減塑、海洋警訊。  
□ 導覽列所有模式都能切換，沒有 JavaScript 錯誤。  
□ 每一段都有 GPT Image 2 生成的多格漫畫，不能只有單張概念圖。  
□ 第 1 段至少 5 格，第 7 段至少 4 格，其餘段落至少 6 格。  
□ 第二段必須包含 1992、貨輪風暴、北上、南下、洋流高速公路、海面深海占領。  
□ 每段都有原文、白話摘要、關鍵詞、短骨架、完整骨架。  
□ Comic Mode 可用圖片 + caption 支援圖像背誦。  
□ Restore Mode 能從短骨架、句首提示、填空、完整遮蔽逐步還原原文。  
□ Quiz Mode 至少 16 題，且包含選擇、填空、排序、看提示猜段落、簡答。  
□ 答錯時會顯示正確答案、來源段落、簡短說明與跳回複習按鈕。  
□ Test Mode 會依照段落長短調整遮蔽程度。  
□ Home Page 有「給家長的話」摺疊區塊。  
□ Parent Mode 有每段家長提問、提示順序、白話轉原文對照。  
□ localStorage 在重新整理後能保留進度。  
□ Mobile 375px 寬度下可正常閱讀與操作。  
□ Desktop 1280px 寬度下版面不過度拉長。  
□ 所有按鈕點擊都有明顯視覺反饋。  
□ 沒有任何外部 CDN、圖片 API 或套件依賴。  
□ 整份 HTML 可以單獨開啟，不需要 server。

如果有任何項目做不到，請在輸出 HTML 前先說明哪一項做不到，以及原因。

## 十三、如果遇到 token 或圖片限制

請按以下優先序輸出：

1. 必輸出：HTML 結構 + CSS + 七段資料 + 首頁 + Story Mode + Read Mode + Comic Mode + Restore Mode。
2. 次優先：Parent Mode + Test Mode。
3. 可後續迭代：Quiz Mode + Order Mode + Retell Mode + Challenge Mode。
4. 圖片若無法生成，請先用清楚的 base64 placeholder 或 SVG placeholder，並保留 `imagePrompt` 與 `imageBase64` 欄位，方便後續替換。

請在輸出開頭明確標示：

本次輸出包含哪些模式。  
哪些模式或圖片需要下一次補完。
