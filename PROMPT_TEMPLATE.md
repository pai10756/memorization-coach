# 給 codex 的指令模板

換新課文時，**複製本檔案內的「指令模板」區塊**，把所有 `{{...}}` 佔位符填上，貼給 codex／Claude／其他 LLM 即可。

> 設計原則背景看 [MEMORIZATION_METHOD.md](MEMORIZATION_METHOD.md)。
> 這份文件只負責「該填什麼」。

---

## 使用步驟

1. 決定課文的基本資訊（標題、體裁、單位數）。
2. 把每個單位（句／段）的五項資料準備好：
   - 原文
   - 關鍵詞（含解釋）
   - 白話翻譯
   - 記憶提示（一句話的口訣）
   - 圖片描述（給 AI 生圖用）
   - 4 字標籤（故事模式用）
3. 設計「因果鏈／脈絡鏈」串起所有單位。
4. 設計 Quiz Mode 的語意問題（不照順序問）。
5. 決定主題色票（依課文情境，不要套用上一課的色）。
6. 把模板內的 `{{...}}` 全部填好。
7. 貼給 codex。

```
- 4 字標籤：...
- 圖片描述：...

第 4 個單位「{{原文 4}}」
- 關鍵詞：...
- 白話：...
- 記憶提示：...
- 4 字標籤：...
- 圖片描述：...

{{視單位數量增刪}}

因果鏈／脈絡鏈：
{{標籤 1}} → {{標籤 2}} → {{標籤 3}} → {{標籤 4}}

【互動功能】
請設計成一個 single-page application，包含以下 5 個核心模式（使用者可以切換）：

1. 「故事模式」(Story Mode)
   - 一次顯示所有單位的圖片橫向排列（手機則直向）
   - 每張圖下方標示原文與 4 字標籤
   - 點擊任一張圖會展開該單位的詳細卡片：關鍵詞、白話、記憶提示

2. 「逐{{句 / 段}}學習」(Learn Mode)
   - 一次只顯示一個單位的完整資訊（圖片大圖 + 關鍵詞 + 白話 + 記憶提示）
   - 上一個 / 下一個切換按鈕
   - 進度條顯示目前在第幾個

3. 「遮蔽測試」(Test Mode)
   - 顯示所有圖片，但內容被遮住
   - 提供 3 個難度等級切換按鈕：
     - Level 1：顯示前段，遮後段（例：「{{範例前半}}___」）
     - Level 2：只顯示首字（例：「{{首字}}______」）
     - Level 3：完全遮蔽（_______）
   - 點擊圖片可揭曉答案
   - 答對 / 答錯按鈕讓使用者自評

4. 「打亂順序挑戰」(Quiz Mode)
   - 顯示語意問題（不依原文順序），讓使用者填首字後接出整句／整段：
     - Q1：{{語意問題 1}}？提示「{{首字 X}}」
     - Q2：{{語意問題 2}}？提示「{{首字 Y}}」
     - Q3：{{語意問題 3}}？提示「{{首字 Z}}」
     - Q4：{{語意問題 4}}？提示「{{首字 W}}」
   - 使用者點擊或輸入答案後顯示對錯

5. 「闖關計時」(Challenge Mode)
   - {{總時長，預設 15}} 分鐘倒數計時器
   - 5 個關卡依序解鎖：
     - 關卡 1（{{X}} 分鐘）：看懂故事，讀 3 次
     - 關卡 2（{{X}} 分鐘）：遮住背前半
     - 關卡 3（{{X}} 分鐘）：遮住背後半
     - 關卡 4（{{X}} 分鐘）：合體背誦
     - 關卡 5（{{X}} 分鐘）：看首字提示完整背一次
   - 每關有「完成」按鈕，全部完成後顯示恭喜畫面

{{若課文是長散文／長段落，再加以下三個模式：}}
6. 「首頁」(Home)：說明使用順序、推薦動線、給家長與學生的引導
7. 「關鍵詞骨架」(Skeleton)：把每段抽成關鍵詞列表，先記骨架再記細節
8. 「家長陪背」(Parent)：提問清單、對話範例、判分標準

【視覺風格】
- 整體風格：{{古典 / 海洋 / 田園 / 戰爭等}} + 現代簡潔 UI 的混合
- 主色：背景 {{#色碼}}、文字 {{#色碼}}、強調色 {{#色碼}}、輔助色 {{#色碼}}
  - 範例 1（古典詩）：米白 #F5EFE0 / 深褐 #3D2817 / 朱紅 #A0353A / 金 #C9A961
  - 範例 2（海洋科普）：米白 #F5EFE0 / 深褐 #3D2817 / 海藍 #1F5F7A / 珊瑚橘 #E7A072
- 字體：標題用 Noto Serif TC / Source Han Serif TC，內文用 Noto Sans TC（不用 CDN，使用系統字體 fallback）
- 課文字級：手機至少 24px，電腦至少 32px
- 按鈕高度 ≥ 42px
- 每個區塊切換要有平滑過渡動畫（淡入 + 微上移，0.3s 內）
- 按鈕要有明顯的點擊反饋
- 整體要適合小五生使用，不要太花俏但要有 {{該課文情境}} 的韻味

【圖片生成】
請使用內建的 image generation 能力，按照上方各單位的「圖片描述」生成 {{N}} 張 {{水彩 / 該課文風格}} 的圖片，並將圖片以 base64 嵌入 HTML（讓檔案完全可攜，傳給別人也能用）。
所有圖片風格必須統一：{{水彩、淡雅、___ 風格、不要太鮮豔、不要寫實照片風}}。

【技術要求】
- 單一 HTML 檔案，所有 CSS、JS、圖片 base64 都內嵌
- 不依賴外部 CDN（可離線使用）
- Mobile-first 設計，但桌面版也要好看（responsive）
- 不需要使用 React 或其他框架，純 vanilla JS 即可
- 加上頁面標題：「《{{課文標題}}》{{第幾段，可省}}背誦小教室」
- 主畫面頂部要有導覽列可切換所有模式
- 使用 localStorage 儲存使用者目前進度，key 開頭用 "memorization-{{課文識別碼}}-"
```

## 已填好的範例：《長恨歌》（二）

以下是給 codex 的完整指令範本，當作填空後長什麼樣子的參考。

```
請幫我製作一個單檔 HTML 互動式背誦工具，主題是白居易《長恨歌》第二段，給台灣小五生背誦使用。

【內容】
背誦目標（共 4 個單位，體裁：詩）：

第 1 個單位「雲鬢花顏金步搖」
- 關鍵詞：雲鬢（像雲一樣美麗的頭髮）、花顏（像花一樣漂亮的臉）、金步搖（走路時會輕輕搖動的金飾）
- 白話：她有美麗的髮型、像花一樣漂亮的容貌，頭上戴著金步搖，非常美麗。
- 記憶提示：楊貴妃很美，打扮很華麗
- 4 字標籤：美麗打扮
- 圖片描述：水彩風格的唐代仕女肖像，黑色高聳髮髻，頭上戴金色步搖髮飾有流蘇，臉部柔美，背景是淡黃色的窗戶與牡丹花，整體柔和典雅。

第 2 個單位「芙蓉帳暖度春宵」
- 關鍵詞：芙蓉帳（華美柔軟的帳子）、暖（溫暖）、度春宵（度過美好的春天夜晚）
- 白話：在華美又溫暖的帳子裡，兩人一起度過春天的夜晚。
- 記憶提示：皇帝很疼她，每晚都很開心
- 4 字標籤：受寵生活
- 圖片描述：水彩風格的古代宮殿臥室內景，粉橘色與金色的芙蓉帳幔半垂下，床帳華美柔軟，床邊有香爐冒著輕煙，整體溫暖朦朧。

第 3 個單位「春宵苦短日高起」
- 關鍵詞：春宵（春天的夜晚）、苦短（覺得太短）、日高起（太陽升得很高了才起床）
- 白話：快樂的春夜一下子就過去了，等到太陽升得很高了才起床。
- 記憶提示：快樂時間太短，太陽高了才起
- 4 字標籤：日高起床
- 圖片描述：水彩風格的古代房間清晨，落地窗外是金黃色的晨光灑進來，陽光照在木質書桌與椅子上，桌上有筆墨紙張，氣氛慵懶寧靜。

第 4 個單位「從此君王不早朝」
- 關鍵詞：從此（從這時開始）、君王（皇帝）、不早朝（沒有一早去上朝處理國事）
- 白話：從那以後，皇帝不再像以前一樣早早去上朝處理國事。
- 記憶提示：皇帝沉迷享樂，開始不早朝
- 4 字標籤：百官空等
- 圖片描述：水彩風格的空蕩龍椅特寫，金色雕花的龍椅高高在上但無人坐，下方有許多戴官帽的大臣背影排列等待，氣氛有些蕭索。

因果鏈：
美麗打扮 → 受寵生活 → 日高起床 → 百官空等

【互動功能】
請設計成一個 single-page application，包含以下 5 個核心模式（使用者可以切換）：

1. 「故事模式」(Story Mode)
   - 一次顯示四張圖片橫向排列（手機則直向）
   - 每張圖下方標示中文句子和 4 字標籤（美麗打扮 / 受寵生活 / 日高起床 / 百官空等）
   - 點擊任一張圖會展開該句的詳細卡片：關鍵詞、白話、記憶提示

2. 「逐句學習」(Learn Mode)
   - 一次只顯示一句的完整資訊（圖片大圖 + 關鍵詞 + 白話 + 記憶提示）
   - 上一句 / 下一句切換按鈕
   - 進度條顯示目前在第幾句

3. 「遮蔽測試」(Test Mode)
   - 顯示四張圖片，但詩句內容被遮住
   - 提供三個難度等級切換按鈕：
     - Level 1：顯示句子前四個字，後三個字遮住（例：「雲鬢花顏___」）
     - Level 2：只顯示首字（例：「雲______」）
     - Level 3：完全遮蔽（_______）
   - 點擊圖片可揭曉答案
   - 答對 / 答錯按鈕讓使用者自評

4. 「打亂順序挑戰」(Quiz Mode)
   - 顯示語意問題（不依詩句順序），讓使用者填首字後接出整句：
     - Q1：哪一句在寫太陽升很高才起床？提示「春」
     - Q2：哪一句在寫楊貴妃很美？提示「雲」
     - Q3：哪一句在寫皇帝不上早朝？提示「從」
     - Q4：哪一句在寫溫暖的春夜？提示「芙」
   - 使用者點擊或輸入答案後顯示對錯

5. 「闖關計時」(Challenge Mode)
   - 15 分鐘倒數計時器
   - 五個關卡依序解鎖：
     - 關卡 1（3 分鐘）：看懂故事，讀 3 次
     - 關卡 2（3 分鐘）：遮住背前兩句
     - 關卡 3（3 分鐘）：遮住背後兩句
     - 關卡 4（4 分鐘）：四句合體背誦
     - 關卡 5（2 分鐘）：看首字提示完整背一次
   - 每關有「完成」按鈕，全部完成後顯示恭喜畫面

【視覺風格】
- 整體風格：唐代古典水彩 + 現代簡潔 UI 的混合
- 主色：米白色背景（#F5EFE0）、深褐色文字（#3D2817）、金色點綴（#C9A961）、朱紅印章色（#A0353A）
- 字體：標題用思源宋體或 Noto Serif TC，內文用 Noto Sans TC（不用 CDN，使用系統字體 fallback）
- 詩句要顯眼，使用較大字級（手機至少 24px，電腦至少 32px）
- 按鈕高度 ≥ 42px
- 每個區塊切換要有平滑過渡動畫
- 按鈕要有明顯的點擊反饋
- 整體要適合小五生使用，不要太花俏但要有古典韻味

【圖片生成】
請使用內建的 image generation 能力，按照上方四個「圖片描述」生成四張水彩風格的圖片，並將圖片以 base64 嵌入 HTML（讓檔案完全可攜，傳給別人也能用）。
四張圖風格必須統一：水彩、淡雅、唐代古典、不要太鮮豔、不要寫實照片風。

【技術要求】
- 單一 HTML 檔案，所有 CSS、JS、圖片 base64 都內嵌
- 不依賴外部 CDN（可離線使用）
- Mobile-first 設計，但桌面版也要好看（responsive）
- 不需要使用 React 或其他框架，純 vanilla JS 即可
- 加上頁面標題：「《長恨歌》（二）背誦小教室」
- 主畫面頂部要有導覽列可切換五個模式
- 使用 localStorage 儲存使用者目前進度，key 開頭用 "memorization-changhenge2-"
```

## 給散文／長段落課文的補充

如果這課是散文（如《海洋的殺手》），單位是「段」而不是「句」，且段落比較長：

1. **單位數**：通常 4–7 段，比詩多。
2. **加 3 個模式**：在指令的「【互動功能】」區塊加入：
   - 6. 首頁／導讀
   - 7. 關鍵詞骨架
   - 8. 家長陪背
3. **主題色換**：依散文主題（海洋／田園／戰爭……）選色票，不要套用詩的米白朱紅。
4. **Quiz 問題**：散文的問題可以更貼近「主旨」、「作者用意」、「事件因果」，不只是「哪一段在寫 ___」。

## 給 codex 之後的事

收到 codex 產出的 HTML 後，對照 [MEMORIZATION_METHOD.md](MEMORIZATION_METHOD.md) 的「品質檢查清單」逐項驗收：
- 五個核心模式齊全？
- Test Mode 三段難度都對？
- Quiz 不照順序？
- 字級與按鈕大小達標？
- 主題色與情境呼應，不是直接抄上一課？
- 單檔可開、不依賴網路？
- 手機與桌面都看過？

不符合的，回去改 prompt 或請 codex 修。
